# 决策番茄钟 | Decision Pomodoro

## 项目概述

**决策番茄钟**是一个基于网页的专注工具，将经典番茄工作法与结构化决策流程相结合。它通过强制性的阶段拆分、产出要求和防发散机制，帮助用户在有限时间内做出高质量决策，避免"分析瘫痪"和过度发散。

**核心设计理念**："完成大于完美"——先产出最小可行结论，再迭代优化。

## 核心功能

### 五阶段决策流程

| 阶段 | 图标 | 时长 | 核心任务 | 强制产出 |
|------|------|------|----------|----------|
| **🧩 问题拆分** | 🧩 | 可调 | 将大问题拆分为"今天能完成的、可判断标准"的小任务 | 具体下一步动作（必须填写才能开始） |
| **🎯 执行轮次1** | 🎯 | 可调 | 只做拆分好的任务，不想别的 | 执行产出（必须填写才算完成） |
| **📝 复盘轮次1** | 📝 | 可调 | 是否达到完成标准？原因是什么？ | 复盘笔记 |
| **🎯 执行轮次2** | 🎯 | 可调 | 继续深入或调整方向 | 执行产出（必须填写） |
| **📝 复盘轮次2** | 📝 | 可调 | 输出主结论/概念卡/证据表/最小行动 | 最终复盘 + 最弱结论（如未达标） |

**默认时长**：65分钟（5+25+5+25+5），可通过时间设置自定义（支持保存最多5套配置）

### 防发散机制

1. **拆分阶段强制产出**：必须先写出"具体下一步动作"才能开始计时
2. **执行阶段强制产出**：没有产出不算完成一轮
3. **10分钟走神检查**：执行阶段10分钟时弹出确认弹窗——"还在主线上吗？"
4. **卡住处理**：提供三种应对策略模板（拆小、换角度、最小行动）
5. **强制止损**：连续两轮未达标时强制输出"最弱结论"并结束

### 辅助功能

| 功能 | 说明 |
|------|------|
| **💭 想法拦截** | 执行中突然想到其他事？记录下来继续执行，不中断当前流程 |
| **📋 今日记录** | 自动保存每轮次的阶段产出、时间、任务名 |
| **📅 导出ICS** | 将记录导出为 `.ics` 文件，拖入 Google Calendar 即可导入 |
| **📝 导出Markdown** | 自动生成结构化的 Markdown 报告，含阶段表格、复盘笔记、完成情况 |
| **⏸ 暂停/继续** | 随时暂停，恢复后继续计时 |
| **🏁 提前结束** | 可手动结束本轮，已完成的阶段会保存 |
| **⚙️ 时间自定义** | 支持保存最多5套时间配置，随时切换 |
| **🔊 音效提醒** | Web Audio API 木鱼音色合成（ConvolverNode 混响空灵版），阶段开始/完成/倒计时提醒，音量可调 |
| **📅 Google Calendar** | 一键打开 Google Calendar 预填事件页 + 下载 .ics 文件 |
| **🤖 AI 助手** | OpenAI 兼容 API，拆分阶段帮拆任务、复盘阶段帮总结 |

## 使用场景

- **论文写作**：拆分论点 → 执行写作 → 复盘逻辑 → 第二轮深化
- **代码设计**：拆分模块 → 执行编码 → 复盘设计 → 第二轮优化
- **决策分析**：拆分选项 → 执行调研 → 复盘证据 → 第二轮验证
- **问题诊断**：拆分症状 → 执行排查 → 复盘假设 → 第二轮确认

## 产出类型

每个阶段可选择产出类型（通过胶囊按钮组快速切换），系统会根据阶段自动推荐默认类型：

| 类型 | 图标 | 默认阶段 | 说明 |
|------|------|----------|------|
| **主结论** | 🎯 | 执行阶段 | 当前阶段的核心发现 |
| **概念卡** | 💡 | — | 定义关键概念，消除歧义 |
| **证据表** | 📊 | 复盘阶段 | 支持/反对的证据清单 |
| **最小行动** | ⚡ | 拆分阶段 | 不追求完美，先动起来 |

切换产出类型时，输入框的 placeholder 示例文字会自动更新为对应类型的格式。

- **JavaScript**：
  - 计时器引擎（setInterval，精确到秒）
  - 阶段状态机（5阶段顺序推进）
  - localStorage 数据持久化（按日期存储）
  - 动态UI更新（阶段指示器、进度条、状态消息）
  - 弹窗系统（卡住、走神、止损、导出、设置）
  - ICS/Markdown 生成器
  - Web Audio API 木鱼音效合成（ConvolverNode 混响）
  - Google Calendar URL Scheme 集成
  - OpenAI 兼容 API 调用（AI 任务拆分/复盘总结）
  - 产出类型胶囊按钮组（事件委托 + placeholder 联动）

### 设计决策记录

| 决策 | 理由 |
|------|------|
| 纯前端，不联网 | 保护隐私，随时可用，零部署成本 |
| localStorage 存储 | 简单可靠，无需后端，数据留在本地 |
| 单HTML文件 | 极简部署，复制到任何地方都能运行 |
| 5+25+5+25+5 阶段 | 经典番茄钟25分钟执行 + 5分钟短休息，两轮确保深度 |
| 强制产出才能继续 | 避免"假装在工作"，确保每阶段都有 tangible output |
| 10分钟走神检查 | 基于经验：10分钟是注意力开始漂移的关键节点 |
| 强制止损机制 | 防止在无效路径上无限投入，接受"最弱结论"也是产出 |
| 时间可自定义（v2） | 不同任务需要不同时长，配置档案系统支持快速切换 |
| 音效用 Web Audio API（v2） | 无需外部音频文件，零依赖，程序化生成音调 |
| 木鱼音色 + ConvolverNode 混响（v3） | 空灵沉浸感，6层音频合成，音量可调 |
| AI 用 OpenAI 兼容 API 格式（v3） | 通用性最强，支持 OpenAI / Claude proxy / 本地模型 |
| Google Calendar URL Scheme（v3） | 零依赖，浏览器直接打开预填事件页，无需 Google API |
| 设置变更时计时器运行中禁用 | 避免中途修改导致状态混乱 |
| 产出类型用胶囊按钮组（v4） | 比下拉框更快捷，一眼看到所有选项，点击即切换 |
| placeholder 按产出类型联动（v4） | 引导用户按对应格式填写，降低认知负担 |

## 使用方式

1. 双击 `pomodoro-v4.html` 在浏览器中打开
2. 在"产出"区域选择产出类型（点击胶囊按钮），填写内容
3. 点击"▶ 开始"启动第一阶段（拆分）
4. 按阶段推进，每阶段必须填写产出才能继续
5. 完成全部5阶段后，导出到日历或 Markdown
