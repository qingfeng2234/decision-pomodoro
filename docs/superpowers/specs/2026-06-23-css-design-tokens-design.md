# 设计：CSS 设计令牌地基重构

- **日期**：2026-06-23
- **范围**：`decision-pomodoro/src/style.css`
- **状态**：已通过设计评审，待写实现计划

## Context（为什么做）

`style.css`（约 1090 行，手写）目前所有颜色、圆角、阴影都是**写死的字面量**，
同一个值（如主紫 `#667eea`）散落在数十处。这带来两个问题：

1. 想统一调整外观（改色、调圆角）必须全文件搜索替换，易遗漏、易出错。
2. 后续想加的**深色模式**几乎无法落地——没有集中的"颜色角色"可供覆盖。

本次目标是打**地基**：把重复的设计值抽成 CSS 自定义属性（设计令牌），
集中到 `:root`。**本次不改变任何可见外观**，只为以后的深色模式 / 视觉调整铺路。

明确**不在本次范围**：深色模式、交互改动、布局调整、HTML / JS 改动。

## 目标与非目标

**目标**
- 在 `style.css` 顶部新增 `:root` 令牌块（颜色 / 圆角 / 阴影 / 过渡）。
- 将文件中等于这些值的字面量替换为 `var(--…)`，严格 1:1 映射。
- 渲染结果**逐像素不变**。

**非目标（YAGNI）**
- 不加深色模式（但令牌结构要让它以后只需覆盖 `:root` 即可实现）。
- 不令牌化间距 / 字号（深色模式不需要，抽取面大、收益低、易错）。
- 不令牌化极少出现的一次性值（见下"保留字面量"）。

## 令牌目录

命名约定：`--c-*` 颜色、`--r-*` 圆角、`--shadow-*` 阴影。
颜色按**角色**语义命名（而非长相），以便深色模式按角色覆盖。

### 颜色

```css
/* 品牌 */
--c-brand: #667eea;   --c-brand-strong: #5a6fd6;   --c-brand-deep: #764ba2;
/* 文字 */
--c-ink: #1e3a5f;     --c-muted: #5c7a9a;          --c-faint: #94a8be;
/* 背景 / 面 / 线 */
--c-bg: #ffffff;      --c-surface: #f8f9fa;        --c-surface-2: #e9ecef;
--c-border: #c8d6e5;  --c-border-soft: #e0e6ef;
/* 强调（AI 浅蓝） */
--c-accent-bg: #f0f4ff;   --c-accent-border: #c5d0f5;
/* 状态：主色 / 浅底 / 深字 */
--c-success: #28a745; --c-success-bg: #d4edda; --c-success-ink: #155724;
--c-danger:  #ff6b6b; --c-danger-bg:  #f8d7da; --c-danger-ink:  #721c24;
--c-warning: #ffc107; --c-warning-bg: #fff3cd; --c-warning-ink: #856404;
--c-info-bg: #d1ecf1; --c-info-ink: #0c5460;
```

### 圆角 / 阴影 / 过渡

```css
--r-sm: 6px; --r-md: 8px; --r-lg: 10px; --r-xl: 12px; --r-2xl: 16px; --r-3xl: 20px;
--shadow-card: 0 6px 24px rgba(102,126,234,.18);
--shadow-lg:   0 20px 60px rgba(0,0,0,.3);
--overlay:     rgba(0,0,0,.5);
--transition:  .2s;
```

> 注：令牌清单为代表性集合。实现时按文件中**实际出现且重复**的值最终确定，
> 允许在保持"语义命名 + 1:1 映射"前提下微调（例如某个色实际还有近似变体需新增别名）。

## 保留字面量（不令牌化）

- 一次性色：音量滑块 `#2d9c8f` / `#e0b44c`；个别按钮 hover 渐变中段色。
- 罕见圆角：如 `4px` / `5px` / `7px` / `14px` 等仅出现一两次的值。
- 判据：**出现 ≤2 次且在深色模式下不需变化**的值，保留字面量，避免噪音。

## 应用方式

- 仅在 `style.css` 内部，把等于令牌值的字面量替换为对应 `var(--…)`。
- 不触碰：HTML、JS、`@media` 媒体查询、布局相关属性。
- CSS 变量是值的别名，计算结果不变 → 外观一致。

## 验证

1. `cd decision-pomodoro && npm run build` 构建通过（无报错）。
2. `npm run dev` 启动，肉眼对照改动前后界面应**完全一致**；
   重点扫：主按钮、三张胶囊卡（含 active/locked/completed 态）、
   状态条四种色（info/success/warning/danger）、各弹窗、GCal 连接区与事件列表、
   想法拦截器（琥珀色块）、音量滑块。
3. 自查：确认无字面量被错配到不等值的令牌；`:root` 中每个令牌值与其替换的原字面量逐一相等。

## 风险

- 唯一实质风险：某字面量被错误映射到**不等值**的令牌 → 局部颜色 / 圆角变化，
  肉眼即可发现，改回正确令牌即可。无数据 / 逻辑风险。

## 后续（不在本次）

本令牌结构就绪后，深色模式可作为独立一轮实现：新增
`[data-theme="dark"] { … }` 或 `@media (prefers-color-scheme: dark)`，
仅覆盖 `:root` 中的 `--c-*` 令牌即可，无需再改各组件规则。
